 Database Images

            

            

              

                
Bagian ini menggunakan metode scraping untuk mengambil seluruh informasi Self Assessment dari halaman eSPM — mulai dari ID, indikator, deskripsi, lokasi, hingga URL gambar.

                
Data yang berhasil diproses akan disimpan secara lokal ke dalam IndexedDB, sehingga dapat digunakan kembali tanpa perlu mengambil data ulang dari situs.

                
Anda juga bisa melihat pada Tabel Summary jumlah data per Range Tanggal, Jumlah Hari, Jumlah Data, dan juga Status mencukupi atau tidak.

                
Tombol Export CSV berfungsi untuk mendownload IndexedDB ke dalam bentuk file Comma-Separated Values untuk keperluan di luar aplikasi ini

 PDF Extract Data

            


            

              

                
Tab ini berfungsi untuk mengekstrak data temuan dari file PDF hasil Self Assessment BUJT, lalu mengolahnya menjadi daftar gambar temuan yang siap diproses.

                
Proses ekstraksi akan mengambil informasi penting dari file PDF seperti tanggal, STA, indikator, jalur/lajur, hingga URL gambar yang ditemukan di dalam dokumen. 

              

            


            
📝 Komponen Utama

            

              

                
    
                  
  •  laporan-pemenuhan-sa #pdfInput: Input file untuk memilih file PDF Self Assessment (*.pdf).
    • 
                  
  • Table Data View #pdfTable: Tabel hasil ekstraksi dari file PDF yang telah dipilih. Berisi daftar data yang siap diproses.
    • 
                  
  •  Download Img Path #pathDownload: Folder tujuan tempat menyimpan gambar asli yang diambil dari eSPM.
    • 
                  
  •  Extract Img Path #pathExport: Folder tujuan untuk menyimpan gambar hasil overlay + kompresi.
    • 
                  
  •  Susunan Nama File #filenameFormat: Format penamaan file gambar yang dihasilkan, dapat disesuaikan dengan kombinasi field seperti id, lokasi, dsb.
    • 
                

              

            


            
🔘 Tombol  Download & Proses

            

              

                
Saat tombol ini ditekan, maka aplikasi akan melakukan proses sebagai berikut:

                
    
                  
  1. Membaca seluruh baris data dari #pdfTable.
    2. 
                  
  2. Memfilter baris yang memiliki URL gambar yang valid.
    3. 
                  
  3. Mengunduh gambar berdasarkan URL dari situs url.espm.go.id.
    4. 
                  
  4. Melakukan kompresi dan menambahkan overlay teks (tanggal, lokasi, STA, jalur).
    5. 
                  
  5. Menyimpan gambar hasil akhir ke tab  Process Gambar #pathExport dengan format sesuai #filenameFormat.
                  
    6. 
                  
  6. Memperbarui tampilan UI dan menampilkan progress per gambar di WebView.
    7. 
                

                
Catatan: Proses ini hanya dapat berjalan jika aplikasi dijalankan dalam mode desktop (menggunakan EXE).

              

            


            
💡 Tips

            
    
              
  • Gunakan file PDF yang telah diunduh dari sistem eSPM dengan struktur yang valid.
    • 
              
  • Jika ada baris kosong atau tidak memiliki URL, data tersebut akan dilewati.
    • 
              
  • Pastikan koneksi internet aktif untuk mengunduh gambar dari eSPM.
    • 
            

          

        


        

          

            
 Process Gambar

            


            

              

                
Tab ini bertanggung jawab untuk menampilkan proses visualisasi gambar secara real-time yang sedang diproses oleh sistem.

                
Setiap gambar yang berhasil diolah (kompresi + overlay) akan ditampilkan dalam bentuk kartu gambar, sedangkan yang masih dalam proses akan ditampilkan sebagai placeholder loading.

              

            


            
🔧 Komponen Utama

            

              

                
    
                  
  • #progressInfo: Label teks yang menunjukkan status progres proses (misalnya: Proses 5 dari 20). 
    • 
                  
  • #progressBarTotal: Progress bar keseluruhan yang menunjukkan persentase proses secara total.
    • 
                  
  • #resultImages: Container yang menampilkan seluruh card gambar, baik dalam bentuk loading maupun hasil akhir.
    • 
                

              

            


            
🖼️ Tampilan Kartu

            

              

                
Placeholder Card (saat gambar masih diproses):

                

                  

                    

                    

                      
                      
                    

                  

                


                


                
Hasil Card (setelah proses selesai):

                

                  
                  

                    

                      ID 01

                      685+000 | A | Non Lajur
                    

                  

                

              

            


            
🔁 Proses yang Terjadi

            
    
              
  1. Tab ini akan aktif setelah tombol  Download & Proses pada tab sebelumnya ditekan.
    2. 
              
  2. Untuk setiap data yang valid, sistem akan menampilkan placeholder card sementara.
    3. 
              
  3. Setelah proses gambar selesai, placeholder akan digantikan dengan gambar hasil kompresi + overlay.
    4. 
              
  4. Status progres ditampilkan di #progressInfo dan progress bar.
    5. 
              
  5. Jika terjadi kegagalan (gagal download, file rusak, dll), kartu akan menampilkan status failed. 
    6. 
            


            
💡 Tips

            
    
              
  • Pastikan folder output sudah ditentukan dengan benar di tab sebelumnya.
    • 
              
  • Jika menggunakan mode BASE64, pastikan WebView2 dapat memuat data image dari string base64. 
    • 
              
  • Gunakan filter dan pencarian jika jumlah gambar sangat banyak.

 Configurations

            


            

              

                
Bagian ini memungkinkan kamu untuk menyesuaikan pengaturan sistem sebelum memproses gambar, mengatur tampilan teks overlay, serta mengelola database lokal (IndexedDB).

                
Setiap perubahan konfigurasi akan diterapkan secara langsung ke dalam objek config, dan dapat disimpan permanen dengan tombol 💾 Save changes.

              

            


            
 Method Load Image Output

            

              

                
Opsi pathMode menentukan cara pemuatan gambar yang akan ditampilkan ke WebView.

                
    
                  
  • SetWorkingDir: Memuat gambar dari folder relatif tempat aplikasi dijalankan.
    • 
                  
  • LoadFilePath: Memuat gambar dari path lengkap dengan prefix file:///.
    • 
                  
  • BASE64: Mengonversi gambar ke format base64 dan ditampilkan langsung di HTML.
    • 
                

              

            


            
 Stamp Manager

            

              

                
Pengaturan ini memengaruhi tampilan stempel (teks overlay) pada gambar hasil proses.

                

                  

                    
    
                      
  • font: Pilih jenis font yang digunakan untuk teks.
    • 
                      
  • fontSize: Ukuran font teks dalam pixel.
    • 
                      
  • heightRectangle: Tinggi kotak latar teks.
    • 
                      
  • paddingText: Jarak antara teks dan tepi kotak.
    • 
                      
  • maxDim: Ukuran maksimal sisi terpanjang gambar setelah kompresi.
    • 
                      
  • sleep: Waktu jeda antara proses gambar (ms), untuk stabilitas sistem.
    • 
                    

                  

                

              

            


            
 IndexDB Manager

            

              

                
    
                  
  • 🗑️ Hapus DB: Menghapus seluruh isi database lokal (IndexedDB).
    • 
                  
  • 📥 Backup DB: Menyimpan seluruh isi database ke file .json.
    • 
                  
  • 📤 Restore DB: Memuat ulang database dari file .json hasil backup.

 Tentang Aplikasi (About)

            


            
📌 Nama Aplikasi

            
Self Assessment Image Processor

              Aplikasi berbasis desktop dengan antarmuka web, digunakan untuk mengambil, memproses, dan mengelola data visual hasil inspeksi Self Assessment dari situs eSPM BPJT.
            


            
⏱️ Durasi Pengerjaan

            
Proses pembuatan aplikasi ini memakan waktu ± 4 hari kerja, mencakup:

            
    
              
  • Perancangan antarmuka & struktur tab.
    • 
              
  • Pengembangan logika parsing HTML & PDF.
    • 
              
  • Pengelolaan IndexedDB dan komunikasi WebView ↔ AHK.
    • 
              
  • Penambahan fitur download, overlay, dan ekspor gambar.
    • 
            


            
🎯 Tujuan Aplikasi

            
Aplikasi ini dibuat untuk memudahkan pengambilan data temuan lapangan berbasis gambar dari sistem eSPM (https://url.espm.go.id) melakukan overlay informasi penting di atas gambar, dan mengorganisasi data dalam bentuk siap
              ekspor.

            
Aplikasi ini dibuat untuk memudahkan proses:

            
    
              
  • Pengumpulan data gambar dari sistem eSPM.
    • 
              
  • Penyamaan data gambar dengan laporan PDF.
    • 
              
  • Penambahan informasi visual (overlay) pada gambar hasil inspeksi.
    • 
            


            
⚠️ Tantangan Teknis

            
Salah satu tantangan terbesar dalam pengembangan ini adalah:

              🧩 Menyamakan urutan gambar dengan entri PDF

            
    
              
  • PDF Resmi tidak menyertakan ID unik dan urutan entri tidak berdasarkan waktu (datetime), melainkan tersusun manual.
    • 
              
  • Sedangkan data gambar dari eSPM memiliki ID dan timestamp.
    • 
              
  • Solusi yang ditawarkan: mencocokkan data berdasarkan nomor dan isi deskripsi secara manual/semi-otomatis.
    • 
            


            
🔎 Fitur Utama

            
    
              
  • Scraping HTML Source eSPM: Mengambil seluruh data Self Assessment (indikator, deskripsi, lokasi, gambar) langsung dari halaman HTML.
    • 
              
  • Manajemen Database Gambar (IndexedDB): Menyimpan hasil parsing lokal ke IndexedDB agar bisa digunakan ulang tanpa scraping ulang.
    • 
              
  • Ekstrak Data dari PDF: Parsing laporan PDF dan mencocokkannya dengan gambar.
    • 
              
  • Download & Overlay Otomatis: Tambahkan teks overlay seperti tanggal, STA, lokasi.
    • 
              
  • Ekspor Gambar Siap Pakai: Nama file bisa dikustomisasi, tersimpan di folder output. 
    • 
            


            
⚙️ Teknologi yang Digunakan

            
    
              
  • Frontend: HTML, Bootstrap 5, jQuery, SweetAlert2, FontAwesome, DataTables.
    • 
              
  • Backend: AutoHotkey v2 + WebView2 (integrasi HTML ↔ AHK).
    • 
              
  • PDF Parsing: Menggunakan PDF.js.
    • 
              
  • Penyimpanan Lokal: IndexedDB.
    • 
              
  • Komunikasi: window.chrome.webview.postMessage() ↔
                MyWindow.ExecuteScriptAsync()
              
    •
